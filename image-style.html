CANVAS · BAKE THE EFFECT INTO A FILE (for export / OG cards) // Apply the Steve Quinlan duotone to an image and return a Blob. // Use when you need a real PNG/JPEG file (CMS upload, OG card, email). async function applyDuotone(file, { shadow = [0, 70, 74], // #00464a highlight = [251, 251, 226], // #fbfbe2 } = {}) { const img = await new Promise((res, rej) => { const i = new Image(); i.onload = () => res(i); i.onerror = rej; i.src = URL.createObjectURL(file); }); const canvas = document.createElement('canvas'); canvas.width = img.naturalWidth; canvas.height = img.naturalHeight; const ctx = canvas.getContext('2d'); ctx.drawImage(img, 0, 0); const data = ctx.getImageData(0, 0, canvas.width, canvas.height); const px = data.data; for (let i = 0; i < px.length; i += 4) { // luminance (Rec. 601) const lum = (px[i] * 0.299 + px[i+1] * 0.587 + px[i+2] * 0.114) / 255; // mix shadow → highlight by luminance px[i ] = shadow[0] + (highlight[0] - shadow[0]) * lum; px[i+1] = shadow[1] + (highlight[1] - shadow[1]) * lum; px[i+2] = shadow[2] + (highlight[2] - shadow[2]) * lum; // alpha untouched } ctx.putImageData(data, 0, 0); return new Promise(res => canvas.toBlob(res, 'image/png')); } // Example // const blob = await applyDuotone(fileInput.files[0]); // document.querySelector('img').src = URL.createObjectURL(blob);
PHOTOSHOP · MANUAL STEPS 1. Open the image. Convert to Smart Object so it stays non-destructive (Layer → Smart Objects → Convert). 2. Add a Black & White adjustment layer (Layer → New Adjustment Layer → Black & White). Default settings are fine — this gives a clean luminance map. 3. Add a Gradient Map adjustment layer above it (Layer → New Adjustment Layer → Gradient Map). 4. Click the gradient strip and build a 2-stop gradient: • Left stop (shadow) — #00464a • Right stop (highlight) — #fbfbe2 Save as a preset named "SQ Duotone" for re-use. 5. Optional: add a Noise layer above (Filter → Noise → Add Noise, 4–8% gaussian, monochromatic) on a 50% grey layer set to Overlay blend mode at ~30%. 6. Export as PNG or JPEG @ 2400px on the long edge. Apply the card frame + portrait shadow in code, not in Photoshop — keeps the source asset clean.
NOTES All four methods produce the same colour map: shadows pinned to #00464a, highlights pinned to #fbfbe2. CSS/SVG is the right default for a Next.js / WordPress site (no asset processing). Canvas is for when you need a real file (Open Graph cards, email). Photoshop is for art-directed one-offs.

BASE PROMPT (for an AI illustration tool) Minimal technical schematic illustration on a warm cream paper background (#f5f5dc) with a faint 40px grid. Composition shows {N} labelled nodes connected by dashed teal (#00464a) curved arrows. Each node is a small frosted-glass rectangle with a thin grey border, containing a large italic serif numeral (01, 02, 03) and a short uppercase mono label. Add 1–2 small accent dots: one mint (#acedda) ringed in teal, one solid teal. Top-right and bottom-left corners carry tiny mono-spaced metadata captions in mossy teal (#29695b). Style: editorial monograph diagram, lab notebook, restrained, intellectual. No 3D, no shading, no gradients, no decoration beyond what is described. Format: 16:9, vector-clean. — {N} = 4 — Node labels: HYPOTHESIS / EXPERIMENT / EVIDENCE / SYSTEM — Top-right caption: PLATE_06 // FRAMEWORK_MAP — Bottom-left caption: EXPERIMENTATION_LOOP_V2
POST-TREATMENT For best results, generate as SVG (or hand-build in code) — AI image models occasionally garble small text. The diagram is simple enough that recreating in SVG using the same recipe (cream grid + glass cards + dashed arrows + mint accent) gives pixel-perfect output every time and stays on-brand.
